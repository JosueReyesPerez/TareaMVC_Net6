Para utilizar Entity Framework Core es necesario utilizar 
varios paquetes.
	
	EntityFrameworkCore
		Paquete principal.

	EntityFrameworkCore.SqlServer:
		Nos ayuda con las conexiones a sqlServer, 
		si queremos otro DB debemos cambiarlo.

	EntityFrameworkCore.Tools o EntityFrameworkCore.Design
		Tools: harramientas de desarrollo para Visual Studio
		Design: herramientas de desarrollo para VS Code u otro editor.

DBContext.
//Es la pieza fundamental de EntityFramework, representa una seccion en la BD
            //Se encarda de la conexion
            //Ofrece metodos para el CRUD
            //Mapea los modelos
Esta clase la tenemos que inyectar en el proyecto para ser utilizada, por ende 
el parametro de tipo DbContextOptions que lo requiere el padre.


Crear una cadena se conexion.

Inyectamos la instacia de DbContext para poder usar entityframework los controladores.
	builder.Services.AddDbContext<AplicationDbContext>(opt => opt.UseSqlServer("name=TareasMVC"));
	-Especificamos la clase que hereda de DbContext
	-Usamos el atributor opt de la clase DbContextOptions, para inyectar la String Conection.
